Learning Git & GitHub Workshop

Learning

Despite being a BCA student, I didn't know much about GitHub even though it was a necessary tool which a developer needed. Even though many of my friends including me were complete beginners, our respected mentor Kushal Sir taught us from the very basics.

Day 1:

We learned how to install Git from the Git-SCM website. After that, we learned how to open Git, initialize a repository using git init, and saw some basic commands.

Day 2:

We continued with basic Git commands. Our mentor explained commands like git status, git add, and git commit. We also learned to view changes using git diff.

Day 3:

By the third day, we had practiced these commands so much that we could create files, add changes, commit, and view history confidently.

Day 4:

This day focused on branching and merging. We learned to create branches, switch between them, merge them, and handle branch conflicts.

Day 5:

We learned to sign in to GitHub and connect local repositories. We practiced git push and git pull to collaborate online.

Day 6:

The final day was exciting! We explored hosting a website using AI (ChatGPT), Git, GitHub, and Netlify, and deployed a simple personal website.

Remaining Confusions:

  • Resolving merge conflicts
  • Using .gitignore
  • Reverting changes safely
  • Writing clear commit messages
  • Solving branch conflicts
